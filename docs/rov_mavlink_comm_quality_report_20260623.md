# ROV MAVLink 通信质量测试总报告（2026-06-23）

## 1. 测试目的

本次测试用于验证中位机 ROS 端 `sealien_ctrlpilot_mavlinkbridge` 与 STM32 下位机在 UDP/IP 链路上的 MAVLink 通信质量，重点观察 5 分钟连续压力通信下的链路稳定性、收发速率、窗口丢包率、连接状态变化以及各类消息相邻帧间隔。

本报告为总报告模板，当前仅完成 EP101 终端测试；其它 STM32 终端保留测试区域，后续按相同流程补充。

## 2. 问题定位与本轮修复

| 问题 | 定位结论 | 修复处理 | 本轮复测结果 |
| --- | --- | --- | --- |
| STM32 侧曾观察到 `IMU_CALIB#12` 明显低于 ROS 侧发送频率 | ROS 测试节点首个 TX route 使用 0 相位，容易与 bridge 内部 1 Hz 心跳形成短 UDP burst；STM32 测试 TX 任务优先级也不利于下行接收压力测试 | ROS 测试 route 首项也加入相位偏移，默认 `tx_route_stagger_ms=20`；STM32 压力 TX 线程优先级下调到低于 `mav:rx` | STM32 侧 `IMU_CALIB#12` 5 个 detail 窗口累计 `3001/3001`，0.00% loss |
| STM32 侧 5 s/60 s 窗口曾出现偏大的监控损失 | 通信质量日志先打印、后更新窗口基线，串口打印期间产生的帧既不计入当前窗口，也不计入下一窗口 | STM32 质量统计改为窗口开始时在临界区内快照计数和 gap 统计，并立即更新基线，再执行串口打印 | 稳态 detail 窗口 TX/RX 均为 0.00% loss，首个窗口仅保留启动边界损失 |
| STM32 侧曾观察到 ROS 心跳约 0.95 Hz | ROS bridge 心跳按“上次实际发送时间 + 周期”限频，timer 回调偶发晚到会长期漂移 | ROS bridge 心跳改为固定 deadline 推进，晚到后自动回到计划节拍 | STM32 侧 ROS 心跳 5 个 detail 窗口均为 `60/60`，0.00% loss |

- ROS 端：已重新编译 `sealien_ctrlpilot_mavlinkbridge`，`colcon build --packages-select sealien_ctrlpilot_mavlinkbridge` 通过。
- STM32 端：使用已烧录的本轮修复版固件进行测试；本次最终复测前未再修改 STM32 固件。

## 9. 当前问题汇总与分析

1. EP101 在有效 5 分钟 detail 窗口内保持在线，ROS 侧 `link_up/down=1/0`，STM32 侧未出现身份过滤或消息路由过滤丢弃，`drops(identity/not_allowed)=0/0`。

2. 原报告暴露的 `IMU_CALIB#12` 下行损失问题已消除。最新复测中，STM32 侧 `IMU_CALIB#12` 累计 `3001/3001`，`IMU_CLEAR#13` 累计 `3000/3001`，两条 10 Hz 控制/配置类消息均达到接近满速。

3. ROS 端心跳调度修正后，STM32 侧 `HEARTBEAT#00` 由此前的约 0.95 Hz 恢复为 `300/300`。最新窗口中 heartbeat gap 为 `999.98/900/1100 ms`，存在轻微 900/1100 ms 抖动，但在 3 s link timeout 阈值内，且未导致链路状态波动。

4. 首个 60 s 窗口仍可见 0.07%-0.15% 的边界损失，后续 4 个 60 s 窗口均为 0.00%。该现象更符合测试启动、窗口起始相位和 STM32 未重启导致的边界统计影响，不表现为持续通信瓶颈。

5. 串口日志首尾出现 `LINK DOWN` 与本次有效窗口无关：首行来自上一轮 ROS 停止后的历史状态，本轮末尾来自 `timeout` 停止 ROS launch 后下位机检测不到 host 心跳。报告未将首尾边界计入有效测试结论。

## 10. 后续优化建议

1. 在 STM32 测试固件中增加 `mavtest reset_stats` 或测试 session id，使每轮测试可在不重启固件的情况下清空累计 total、link_up/down 和 gap 统计，报告数据会更干净。

2. 对 heartbeat jitter 有更严格要求时，可继续将 ROS bridge heartbeat 放入独立高优先级 callback group 或独立发送线程；当前 900-1100 ms 抖动不影响 1 Hz 计数和链路保活。

3. 后续 EP102、EP103 按同一命令和模板补测。多板并发前建议先分别完成单板 5 分钟基线，再执行多端同时压力测试，便于区分单板问题与总线/主机调度问题。

4. 建议补充自动日志解析脚本，直接从 ROS/STM32 原始日志生成窗口汇总、消息汇总和 gap 表，降低手工整理报告时的口径差异。

## 11. 测试结论

本轮修复后，EP101 的 MAVLink 通信质量达到当前压力测试目标。ROS -> STM32 下行测试消息为 `6000/6000`、0.00% loss；STM32 -> ROS 上行数据为 `42287/42300`、0.03% loss，且后续 4 个 60 s 稳态窗口均为 0.00% loss。

STM32 侧实测也与 ROS 侧一致：上行 `42299/42310`、0.03% loss，下行 `6301/6302`、0.02% loss。高频 50 Hz 的 `IMU_DATA#01`、`DVL_DATA#18`，低中频 10 Hz 的状态/控制消息，以及 1 Hz 心跳均满足当前单板通信质量测试要求。
